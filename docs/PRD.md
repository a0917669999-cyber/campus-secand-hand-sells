# PRD - CampuSwap 校園二手交易平台

**專案名稱**：CampuSwap
**專案描述**：一個專為大學生設計的校園二手交易平台，內建 AI 估價推薦系統與進階搜尋過濾功能，讓校園內的二手物品交易更快速、公平且便利。

---

## 1. 專案概述

### 背景與動機
大學生在每個學期初末常有買賣二手教科書、電子產品與宿舍用品的需求。然而，現有平台多為泛用型，缺乏校園專屬的信任機制與交易便利性。此外，賣家在發布商品時常不知如何定價，買家也難以判斷價格是否合理。

### 目標用戶
- **主要用戶**：校園內的大學生（同時扮演買家與賣家的角色）。

### 核心價值主張
提供一個僅限校園內的高信任度交易環境，並透過「AI 估價推薦」幫助買賣雙方建立公平的定價基準，搭配強大的「搜尋與過濾」功能，大幅降低媒合時間。

---

## 2. 功能需求

以下列出 CampuSwap 的主要功能，特別強調本次核心需求 F-02 與 F-04：

- **F-01 帳號與身份驗證**
  - 作為學生，我希望能夠註冊並登入系統，以便我在平台內進行交易。
- **F-02 AI 估價推薦 (AI Pricing) 【核心】**
  - 作為賣家，我希望在發布商品時，系統能根據物品類別、品牌與狀況（如：九成新）自動推薦合理的價格區間，以便我能快速且合理地定價。
- **F-03 商品發布與管理**
  - 作為賣家，我希望能夠上傳商品照片、填寫描述並發布商品，以便讓買家看到我欲販售的物品。
- **F-04 搜尋與過濾 (Search & Filter) 【核心】**
  - 作為買家，我希望能夠輸入關鍵字搜尋商品，並可依據類別（書籍、電子產品、宿舍用品）、價格區間進行精準過濾及結果排序，以便我能快速找到符合預算的目標物品。
- **F-05 買賣雙方溝通**
  - 作為買賣雙方，我希望能夠透過平台內的簡單聯絡機制確認面交細節，以便順利完成交易。

## 3. 非功能需求

### 技術限制
- **後端框架**：Python + Flask
- **前端技術**：HTML / CSS / JavaScript
- **資料庫**：SQLite
- **AI 整合**：需串接外部 AI 模型 API 進行價格預測與估價。

### 效能與安全考量
- **效能**：搜尋與過濾功能 (F-04) 的結果應在 1~2 秒內載入完畢，確保流暢體驗。
- **安全**：
  - 需實作基本的防護機制以防止 SQL Injection 與 XSS 攻擊。
  - 使用者密碼應雜湊加密儲存。

---

## 4. MVP 範圍

### Must Have (必須具備)
- 商品發布功能（含手動填寫價格與上傳資訊）。
- 基本關鍵字搜尋 (F-04)。
- 分類與價格過濾 (F-04)。
- 基礎的介面與 SQLite 資料庫連接。

### Should Have (應該具備)
- AI 估價推薦系統 (F-02) 整合至商品發布流程。
- 搜尋結果排序（如：依價格高低、最新發布等）(F-04)。
- 基本使用者註冊與登入功能。

### Nice to Have (有會更好)
- 站內即時通訊機制。
- 商品收藏/追蹤功能。
- 評價與個人信用系統。
